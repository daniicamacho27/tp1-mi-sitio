# Reflexion:
# 1. ¿Qué diferencia hay entre Scrum y Kanban?
Scrum y Kanban son metodologías ágiles, pero se organizan de manera diferente. Scrum trabaja con iteraciones llamadas "sprints" que tienen un tiempo fijo y roles definidos, mientras que Kanban se basa en un flujo continuo de tareas sin tiempos establecidos. En Scrum se planifica el trabajo por etapas, mientras que Kanban permite visualizar y mover las tareas de manera flexible. Kanban se enfocan en limitar el trabajo en progreso y mejorar el flujo de tareas.
# 2. ¿Cuál te parece más útil para un proyecto individual y por qué?
Para un proyecto individual considero que Kanban es más útil, porque permite organizar las tareas de manera simple y flexible. No es necesario trabajar con tiempos fijos ni roles, lo que facilita adaptarlo al ritmo personal. Además, permite visualizar claramente qué tareas están pendientes, en proceso y terminadas. Esto ayuda a mantener el orden y seguir el progreso del proyecto de forma clara.
# 3. ¿Qué es un MVP (Producto Mínimo Viable) y cómo lo aplicarías si te piden hacer un sitio web para un negocio local?
Un MVP (Producto Mínimo Viable) es una versión inicial de un producto con las funciones básicas necesarias para que pueda utilizarse. En el caso de un sitio web para una empresa local, el MVP incluiría las páginas principales, como inicio, servicios y contacto.Se debe organizar priorizando las funcionalidades más importantes y dejando mejoras para etapas posteriores. Primero se desarrolla lo esencial y luego se agregan mejoras como diseño avanzando o nuevas secciones.
# 4.¿Cómo te ayudó el tablero Kanban a organizar las tareas de este TP? Mencioná un ejemplo concreto. 
El tablero Kanban me ayudo a organizar las tareas del trabajo práctico de forma visual y ordenada. Me permitió ver claramente qué tareas debía hacer, cuáles estaban en progreso y cuales ya había terminado. Esto facilitó el seguimiento del avance del proyecto y evitó olvidarme tareas. También me ayudó a dividir el trabajo en partes más pequeñas y manejables. Ejemplo: Al organizar las tareas como crear la estructura HTML,aplicar estilos CSS y creae el Pull Request. Primero las ubique en "Backlog", luego pasaron a "In Progress" mientras las realizaba y finalmente a "Done" cuando las termine. Esto me permitió llevar un control claro del avance del trabajo práctico. Gracias a esta organización, pude completar el TP de manera ordenada.
